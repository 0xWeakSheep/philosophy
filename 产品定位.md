# 产品定位文档：意识形态镜室

## 一句话定位

一个以哲学分析框架为核心引擎的 AI 自我认知产品——不诊断你，不教育你，只帮你看见自己相信什么、在乎什么、卡在哪里、能往哪走。

## 我们要解决什么问题

现有的自我认知工具大多落在两个极端。一端是心理测评和情绪日记——轻量、可及，但停留在症状和行为表层，触及不到一个人默认的世界观和意义结构。另一端是心理咨询和治疗——深度足够，但门槛高、成本大、依赖人力，规模化困难。中间地带被泛泛的“AI 陪伴”和“人生教练”类产品占据，它们能聊，却缺乏一套严谨的认知框架来驱动对话的方向和质量——聊得越久，用户越容易在原地打转。

更深层的问题是：大多数工具在帮用户**管理问题**，而不是帮用户**理解自己为什么会制造这些问题**。一个反复陷入相似关系困境的人，需要的不是第十份情绪记录，而是一次对自己“场域论”的暴露——你默认相信的关系规则是什么？你假定的“正常”从哪来的？这不是心理学的语言，却比心理学更前置：它追问的是心理学范式本身无法追问的预设。

## 核心洞察：为什么是4×4

未明子的“主义主义”框架提供了四个维度和四种要素，原本是用来解码外部意识形态的工具。但它的真正力量在于**反转方向**——每一个维度都可以成为向内追问的入口：

**场域论 → “我的世界是什么样的？”** 你无意识携带的操作系统——你默认相信什么规则、什么边界、什么“当然如此”。

**本体论 → “对我来说什么是真的、不可妥协的？”** 在你的世界里，什么东西具有坚不可摧的存在性——是理性、是关系、是某种信念、还是尚未被命名的东西。

**现象论 → “我如何体验这一切？”** 你的感知滤镜——情绪、直觉、理性在你身上如何分布，你与自己的感受之间隔着什么。

**目的论 → “我到底为了什么？”** 你现在的行为最终指向哪里，那个终点是你选的还是被塞给你的。

四种要素——秩序（1）、冲突（2）、中心（3）、虚无（4）——则对应**自我认知的深度标尺**：你在无意识循环中（1），还是在撕裂中挣扎（2），还是找到了某种脆弱的平衡（3），还是敢于面对根本的不确定性并从中生出真正的自由（4）。

这意味着每个用户都可以被“编码”——不是被定型，而是被定位。而且这个编码会随使用过程演变。这不是静态的人格分类，而是动态的意识形态地图。

## 产品隐喻：意识形态镜室

你走进去，面对的不是一面镜子，而是四面——分别对应场域、本体、现象、目的。每一面镜子反射的都不是你的外貌，而是你相信什么、在乎什么、怎么感受、去向哪里。

AI 不是告诉你“你是什么”的人——它是那个调整镜子角度、递给你手电筒的人。最终的发现权，永远在用户自己手里。

## 产品如何工作：三层交互模型

### 第一层 · 入口：低门槛触发

用户通过轻量的、直觉式的交互暴露自己的认知倾向。可能的形态包括叙事场景选择（在几个虚构情境中做选择）、卡牌式自我描述（摆放四个维度的直觉排序）、或简短的自由书写。这一步的核心要求：有趣、不吓人、五分钟内完成、用户不觉得自己在被“分析”。

产品在这一层默默记录用户的选择模式，作为后续深度对话的起点。

### 第二层 · 核心：框架驱动的深度对话

AI 进入苏格拉底式的追问模式。但它不是普通的聊天机器人——它背后运行着4×4分析引擎，知道在什么时机切入场域论的追问、什么节点该推向本体论的悬崖、什么时候用户准备好了面对目的论的不适。

关键设计原则：AI 不给出答案，它追问。用户对自己编码的发现必须是**自己做出的**——被告诉的认知没有粘性，自己挖出来的才会留下。

### 第三层 · 显影：可视化的自我地图

探索结束后，用户看到的是一个动态的“自我意识形态地图”——一个4×4矩阵上亮起的坐标点，附带演化路径。它告诉用户：你当前的位置（如1-2-3-1）意味着什么，你的盲区在哪里，如果要移动到你想要的位置，需要面对什么。

更重要的是，这个地图是**版本化的**——用户可以回到这个产品，看到自己的编码如何随时间演变。自我认知不是一个答案，而是一条轨迹。

## 我们与谁不同

**不是心理健康产品。** 我们不诊断、不治疗、不提供临床建议。我们追问的是比症状更底层的认知结构。

**不是人格测评。** MBTI 和九型人格给出静态标签，我们给出动态坐标和演化路径。标签让人安心，坐标让人行动。

**不是通用 AI 聊天。** 对话背后有严谨的哲学框架驱动方向和深度，而非依赖模型自身的“共情”能力。

**不是哲学教育。** 用户不需要懂未明子、拉康或康德。框架是引擎，不是课程内容。用户只感受到追问的质量，感受不到理论的存在。

## 目标与非目标

### 目标

- 让用户在30-60分钟内完成一次从触发到显影的完整自我探索旅程

- 让用户带走一个可回溯、可演化的自我认知坐标

- 建立一种“值得回来”的产品关系——不是为了打卡，而是为了看见变化

### 非目标

- v1不做社交功能和社区

- 不做心理测评报告或临床建议

- 不做内容消费（不推送文章、课程、解读）

- 不是未明子粉丝专属产品——框架是引擎，产品面向任何有自我探索需求的人

## 风险与开放问题

**框架的“暴力”问题。** 4×4框架本身带有未明子个人哲学立场的强烈预设（尤其是“4优于1”的线性进步叙事）。产品是否需要、能否将框架去政治化，保留其结构力量而剥离其价值判断，是核心设计挑战。

**深度与可及性的张力。** 一个有哲学框架驱动深度的产品，如何不让用户在入口层就被吓退——这是第一层交互设计的成败关键。

**“知道”之后呢？** 用户看见了自己的编码，然后呢？认知到行动之间有一段产品目前没有设计覆盖的地带。这不是 v1必须解决的问题，但需要在产品叙事中被诚实对待。

**AI 追问的质量边界。** 当前的 LLM 在苏格拉底式追问中能否持续保持方向感和深度，而不是滑向泛泛的共情或说教——这需要大量的 prompt 工程和测试验证。

**语言模型的文化预设。** 4×4框架植根于欧陆哲学传统，而主流 LLM 的训练语料以英语世界的分析哲学和实用主义为主。两者之间可能存在追问方向和价值判断上的隐性冲突。

## 下一步建议

1. 选定一个最小可验证的核心场景（建议从“关系困境”切入——用户动机最强，场域论追问最自然），设计一个从触发到显影的完整 demo 流程

2. 用真人跑通这个流程，验证“用户是否真的能通过框架驱动的对话发现自己之前不知道的东西”

3. 根据验证结果决定产品形态（对话应用 / Web 工具 / 其他）和技术选型

4. 明确框架的“去政治化”策略——保留什么、剥离什么、改什么
